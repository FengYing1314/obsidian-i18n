title: "快速开始：翻译你的第一个 Obsidian 插件"
sidebarTitle: "快速开始"
description: "按照七个步骤完成第一次插件字符串提取、AI 翻译、审阅与应用。"
---

本指南会带你从零开始翻译一个已经安装在 Obsidian 中的社区插件。完成后，你会得到一份已经提取、翻译、审阅并应用到实际界面的译文。

<Steps>
  <Step title="打开 i18n 面板">
    点击 Obsidian 侧边栏中的地球图标，或在命令面板里执行用于打开 i18n 面板的命令。管理中心会随即打开。
  </Step>
  <Step title="配置 AI 服务商">
    第一次翻译前，请先进入 **设置 → 社区插件 → I18N → 语言模型**，填写 API Key 和模型信息。

    详细支持列表见 [AI 服务商](/configuration/ai-providers)。
  </Step>
  <Step title="选择要翻译的插件">
    在管理中心的插件列表中找到目标社区插件，点击它进入翻译工作区。
  </Step>
  <Step title="提取可翻译字符串">
    点击 **提取 / Extract**。AST 引擎会扫描插件的编译产物，并抓出按钮、提示、设置项等面向用户的文本，不会修改源文件。
  </Step>
  <Step title="使用 AI 批量翻译">
    点击 **AI 翻译 / Translate**。插件会把提取结果按批次发送到你已配置的服务商。

    <Tip>
      正式发送请求前，面板会显示预计 Token 数和费用。确认无误后再开始，能够避免不必要的支出。
    </Tip>
  </Step>
  <Step title="在可视化编辑器中审阅">
    翻译完成后，你会看到原文和译文并排显示。点击任意译文单元格即可修改，离开单元格后自动保存。
  </Step>
  <Step title="应用译文">
    点击 **应用 / Apply**，把译文注入到插件运行文件里。

    如果应用后界面异常，先点击 **还原 / Restore** 恢复原始状态，再排查具体字符串问题。
  </Step>
</Steps>

<Note>
  这里提到的 `language`、`llmLanguage` 等字段表示**插件输出译文**的目标语言，例如 `zh-cn`、`ja`、`fr`。它们和当前文档站使用的语言切换是两套不同的概念。
</Note>

<Tip>
  如果你需要更完整的流程，包括主题翻译、译文分享和自动化能力，请继续阅读 [翻译一个插件](/guides/translate-a-plugin)。
</Tip>
