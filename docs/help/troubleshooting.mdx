## AI 连接与接口问题

这些问题对应 **语言模型** 标签页里的 **测试连接**。

### 401 Unauthorized

当前 Profile 的 API Key 无效，或粘贴时带了空格。删除后重新填写，再执行一次测试。

### 404 Not Found

当前接口地址不正确。优先检查 Base URL 是否写成了服务商要求的 API 根路径。

### 429 Too Many Requests

说明当前账号触发了频率限制或额度限制。先降低 `llmConcurrencyLimit`，再重试。

### Timeout

把编辑器 AI 面板中的 `llmTimeout` 调高，或切换到响应更快的模型 / 服务商。

### Network / SSL error

如果问题发生在 GitHub 原始资源下载，先切换 **综合设置 → 加速节点**。如果问题发生在 AI 服务连接，检查当前 Base URL 是否可达。

## 提取不到内容

### 插件提取不到字符串

插件提取依赖目标插件目录里的 `main.js`。如果 `main.js` 不存在，AST 提取不会生成结果；这时可以检查插件安装是否完整，或改用 Regex 结果排查。

### 主题提取不到字符串

当前主题提取器只读取 `theme.css` 里的 `/* @settings */` 块。如果主题没有这个块，或块里没有 `name`、`title`、`description`、`label`、`markdown` 字段，就不会生成条目。

## 翻译结果问题

### 模板变量如 `${name}` 丢失

先在编辑器中搜索对应原文，再把缺失的占位符补回去。应用前先确认变量、括号和转义字符没有被改坏。

### 点击应用后提示“插件重载失败”

插件应用失败时，当前实现会自动执行回滚。先回到编辑器，检查最近修改的译文行，再重新应用。

## 安装与导入问题

### 用 BRAT 安装后看不到插件

BRAT 只负责下载文件，不会自动启用插件。请到已安装社区插件列表里找到 **I18N** 并手动打开。

### 手动安装后插件加载失败

请确认目录结构严格如下：

```text
.obsidian/plugins/i18n/main.js
```

同时还需要有 `manifest.json` 和 `styles.css`，并且文件夹名必须是 `i18n`。

### 本地导入失败

管理中心 **管理** 标签的导入入口优先对应管理导出的 `.i18n.gz` 文件。若你导入的是 `.json`，先确认它来自旧版管理导出格式。
