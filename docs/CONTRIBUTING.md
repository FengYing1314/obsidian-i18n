# 参与 Obsidian i18n 文档维护

本指南说明如何在当前仓库里维护 Mintlify 文档，并确保中英文页面始终和源码一致。

## 本地预览

从 `G:\goProjects\obsidian-i18n\docs` 目录运行：

```bash
npx mint dev
```

预览前请确认你就在 `docs/` 目录，因为 `docs/docs.json` 位于这里。

## 提交前必须执行的检查

```bash
npx mint validate
npx mint broken-links
```

## 新增或修改公开页面时的规则

1. 先改中文主文页面。
2. 再同步 `docs/en/` 对应英文页面。
3. 如有新增页面，同步更新 `docs/docs.json` 导航。
4. 重新执行 `validate` 和 `broken-links`。

## 文档内容规则

- 只写已经在源码或已验证行为中出现的事实
- 不要写不存在的英文 UI 标签名
- 真实 UI 标签若是中文，英文页也直接写中文标签并使用反引号包裹
- 不要把插件译文语言码和文档站语言配置混为一谈

## 最小术语表

- 产品名：`Obsidian i18n`
- 插件显示名：`I18N`
- 设置页标签：`综合设置`、`RE 配置`、`AST 配置`、`沉浸翻译`、`语言模型`、`共建生态`
- 管理中心标签：`插件`、`主题`、`管理`、`自动化`、`鸣谢`
- Cloud 视图标签：`社区目录`、`探索资源`、`管理中心`

## 不应写入文档的内容

- 通过猜测补出来的按钮名、菜单路径、命令名
- 没有源码或验证依据的后端支持说明
- “通常 / 大概 / 可能 / 推荐 / 适合 / 16+ / JSON pack” 这类会把事实说虚的表述

## 推荐的修改顺序

1. 查相关源码文件和本地化文本。
2. 改中文页面。
3. 改英文镜像。
4. 跑 Mintlify 校验。
5. 再检查断链与导航。
